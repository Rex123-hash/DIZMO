# Submission Compliance Checklist

This checklist is based on the Slack Agent Builder Challenge official rules pasted into the workspace.

## Track

Submit DIZMO to:

> Slack Agent for Good

Reason:

- DIZMO addresses disaster response and public safety coordination.
- This track does not require Slack Marketplace submission.
- This track requires the project to be newly created during the submission period.

## Critical Requirements

- Project must use at least one required technology:
  - Slack AI capabilities
  - MCP server integration
  - Real-Time Search API
- DIZMO should use MCP server integration and a search adapter, with Slack AI style summarization and extraction.
- Project must be installable and run consistently in the intended Slack environment.
- Demo video must show the working project, not just slides.
- Demo video should be less than 3 minutes.
- Demo video must be public on an accepted video platform.
- Submission must include an architecture diagram.
- Submission must include the Slack developer sandbox URL.
- The sandbox must grant access to:
  - `slackhack@salesforce.com`
  - `testing@devpost.com`
- Submission materials must be in English.

## Safety and Content Constraints

DIZMO must not include:

- Real emergency victim data.
- Confidential, proprietary, or sensitive information.
- Third-party copyrighted music.
- Third-party trademarks or logos without permission.
- Security exploit details.
- Malware, spyware, or harmful code.
- Non-public source code, SDKs, APIs, or libraries.

Use fictional disaster data and demo fixtures only.

## Judging Strategy

Stage One is pass/fail viability:

- DIZMO must clearly fit the submission requirements.
- The app must function as shown in the video.
- The Slack sandbox must be accessible to judges.

Stage Two is equally weighted:

- Technological Implementation
- Design
- Potential Impact
- Quality of the Idea

Tie-breaking starts with Technological Implementation, so code quality and credible architecture matter.

## Prize Eligibility Note

A winning submission can only win one prize. First and second place winners are not eligible for achievement prizes.

This means the priority is winning the Slack Agent for Good track, while still being strong enough for Best Technological Implementation or Best UX if the main track is missed.

## Build Implications

- Keep the demo deterministic.
- Use fictional data everywhere.
- Add audit logs and human approval to strengthen trust.
- Do not overclaim emergency-service capability.
- Make the project easy for judges to install and test.
- Ensure the video and app behavior match exactly.
